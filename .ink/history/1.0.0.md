# 1.0.0

> 2026-03-29 — init

## What changed

Initial public release of create-ink — an AI-native project scaffold with structured versioning, changelogs, and agent memory.

## Decisions

- **Single AGENTS.md** over per-platform config files (.cursorrules, CLAUDE.md, copilot-instructions.md). All modern AI tools converge on reading AGENTS.md. One file to maintain.
- **`.ink/` hidden folder** for CLI and history files. Only `ink.config.json` and `AGENTS.md` visible at root. Minimal pollution.
- **One markdown file per version** instead of one folder with multiple files. One read call = full picture. Agents include relevant sections, skip the rest.
- **Shell-based commit validation** over commitlint. Our commit-msg hook does the same job with zero dependencies. Removes ~100 packages and a deprecation warning.
- **Three git hooks**: `commit-msg` (validates format + history file), `pre-push` (catches --no-verify bypasses), `post-commit` (auto-pushes).
- **Zero dependencies** for the scaffold package itself. Uses only Node.js built-ins. Target projects get one devDependency (husky).
