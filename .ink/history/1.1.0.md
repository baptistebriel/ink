# 1.1.0

> 2026-03-30 — feat

## What changed

Added BRAIN.md — a living project memory file that agents update every session. Complements AGENTS.md (rules) with persistent state (decisions, progress, context, learnings).

New features:
- **BRAIN.md** template created during scaffold — agents read it at session start, update it before committing
- **`context` command** (`node .ink/cli.js context`) — outputs current version, latest history file, and full BRAIN.md content in one call. The "agent handoff" command for session start.
- **`pre-commit` hook** — reminds agents to stage BRAIN.md (warn only, never blocks)
- **Enhanced `status` command** — now shows BRAIN.md state with last-modified timestamp

## Why

ink governs *how* agents commit (conventional commits, history files). But there was no governance for *what agents remember* between sessions. AGENTS.md is read-only rules. BRAIN.md is living state — the missing half that makes agent handoffs actually work.

## Decisions

- **Named `BRAIN.md` not `CLAUDE.md`** — agent-agnostic naming. Works with Claude, Cursor, Codex, Copilot, any AI tool.
- **Warn, don't block** on BRAIN.md — matches ink's philosophy of hard-blocking on structural things (commit format) but being lenient on behavioral things (documentation). Rapid prototyping stays fast.
- **`context` command outputs everything** — one command, full picture. No need to manually read three separate files at session start.
- **pre-commit is a new hook** — ink had commit-msg/post-commit/pre-push but no pre-commit. Natural gap to fill.
- **BRAIN.md preserved on re-init** — `npx create-ink init` won't overwrite an existing BRAIN.md.
